# 项目审查修复记录

本文件对应 `项目审查.md` 的问题逐项核查和修复。

## 已修复或已改进

1. 早停与学习率调度
   - `train.py` 已加入 `--patience`、`--min-delta`、`ReduceLROnPlateau`、`--lr-patience`、`--lr-factor`。
   - 为了保持现有 `report_final.md` 对应的 40 轮全量实验可复现，早停和学习率调度默认关闭；复现实验时可显式加 `--lr-scheduler --patience 10`。
   - 注意：现有全量实验的最佳验证 loss 实际出现在 epoch 36，而不是审查文件中写的 epoch 12。第 12 轮只是接近最优。

2. `.gitignore`
   - 已新增 `.gitignore`，排除 `__pycache__`、`.venv`、`data/`、`data_cn/`、`runs/`、`logs/`、checkpoint、NIfTI 和 tar 分卷等大文件/生成文件。

3. 输出 clamp 梯度
   - `src/model.py` 已将普通 `torch.clamp` 改为 straight-through clamp。
   - 前向输出仍限制在 `[0, 1]`，反向传播不再因超出范围而直接给零梯度。

4. 训练损失重复计算
   - `train.py` 的 `loss_fn` 现在一次返回 total loss、L1 和 MSE，日志不再重复计算 L1/MSE。

5. raw dict 输出和 print
   - `train.py`、`evaluate.py` 已改用 `logging`，训练日志包含 epoch、train/val loss、学习率、best 标记和早停计数。

6. `caseid` 类型
   - `evaluate.py` 写出 `case_metrics.csv` 前强制 `caseid` 为字符串。
   - `scripts/fill_report.py` 读取病例指标时也使用 `dtype={"caseid": str}`。

7. 数据增强、AMP、梯度裁剪
   - `train.py` 已加入 `--augment`：训练时对 noisy/clean 成对随机水平/垂直翻转。
   - 已加入 `--amp`：CUDA 混合精度训练，默认关闭以保持复现实验与原结果一致。
   - 已加入 `--grad-clip`：默认 `1.0`，可用 `0` 关闭。

8. 硬编码路径
   - `scripts/run_server.sh` 和 `scripts/smoke_server.sh` 的 `ROOT`、`DATA_ROOT` 改为环境变量默认值。
   - 原服务器路径仍是默认值，因此旧命令不受影响。

9. 配置管理
   - 新增 `configs/denoise_full.json`，记录全量实验的数据路径、划分、模型、训练和评估参数。

10. 单元测试
    - 新增 `tests/test_core.py`，覆盖有效切片筛选、全空切片回退、模型 forward shape/range、straight-through clamp 梯度。

11. 报告文件关系
    - `README.md` 已说明 `report_final.md` 和 `report_final.pdf` 是全量实验权威提交版本。
    - `report_final_24case.md`、`report_final_mini.md` 保留为历史子集实验记录。

## 未直接改动的点

1. 2D 模型缺少 3D 上下文
   - 这是当前基线方法的实验局限。改成 2.5D 或 3D U-Net 会改变实验设计和已有全量结果，建议作为报告中的后续改进，而不是在最终提交前临时替换。

2. `scripts/fill_report.py` 模板仍在脚本内
   - 外置模板能提升维护性，但不会影响实验正确性、指标或复现。当前优先修复了训练稳定性、工程卫生和结果可追踪性。

3. 间接依赖锁定
   - `requirements.txt` 保持直接依赖；如需完全冻结服务器环境，可在服务器执行 `.venv/bin/pip freeze > requirements-lock.txt`。

## 验证

```bash
python3 -m py_compile train.py evaluate.py scripts/fill_report.py scripts/prepare_data.py scripts/plot_training_curve.py src/dataset.py src/model.py
python3 -m unittest discover -s tests
```

服务器环境中应使用：

```bash

```
