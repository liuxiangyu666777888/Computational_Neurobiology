# 项目审查：T1 MRI Denoising Experiment

这是一个计算神经学课程的期末作业项目，使用 2D Residual U-Net 进行 T1 MRI 图像去噪。整体结构清晰、可复现性较好，但存在以下改进空间：

---

## 🔴 严重问题

### 1. 训练过拟合，无早停机制 — [train.py:151](train.py#L151)

查看 [metrics.csv](runs/denoise_unet/metrics.csv)，验证 loss 在 epoch 12 达到最低点 (0.00270)，之后在 epoch 40 回升到 0.00275。模型从 epoch 12 到 40 实际上在持续过拟合，浪费了 28 个 epoch 的计算资源。建议添加早停（patience=10）和学习率衰减。

### 2. 无 `.gitignore` 文件

`__pycache__/` 目录随处可见。建议添加 `.gitignore` 排除 `__pycache__`、`.venv`、`runs/`、`data/` 等。

---

## 🟡 代码质量问题

### 3. `train.py` — `forward` 中 `clamp` 会截断梯度 — [src/model.py:56](src/model.py#L56)

```python
return torch.clamp(x + residual, 0.0, 1.0)
```

`torch.clamp` 对于超出 `[0,1]` 范围的值，梯度为零。虽然残差通常很小，但当模型预测超出范围时梯度会丢失。好在 clamp 在 [0,1] 范围内的梯度是 1，实际影响不大，但值得注意。

### 4. `train.py` — 重复计算 L1 和 MSE — [train.py:77-78](train.py#L77-L78)

`loss_fn` 已经计算了 `L1 + 0.2*MSE`，但 `run_epoch` 又分别计算了一次 `l1_loss` 和 `mse_loss`。每次 forward 做了两次 L1 和两次 MSE 计算。对于 logging，可以在 `loss_fn` 中返回三个值，避免冗余。

### 5. `train.py:173` — `print(row)` 直接输出 raw dict

应该使用 logging 模块或格式化的输出。

### 6. `evaluate.py` — `caseid` 解析不健壮

```python
caseid = str(row["caseid"])
if caseid.endswith(".0"):  # line 65 in fill_report.py
    caseid = caseid[:-2]
```

pandas 可能把 `1000187` 解析为 `1000187.0`。这应该在 CSV 读取时就通过 `dtype={"caseid": str}` 解决（`read_cases` 已经做了），但 `fill_report.py` 读取 `case_metrics.csv` 时可能已丢失类型信息。

### 7. 全局使用 `print()` 而非 `logging`

所有 Python 文件使用 `print()` 而非 Python 的 `logging` 模块。对于长时间运行的训练任务，logging 模块提供时间戳、级别过滤等优势。

### 8. `scripts/fill_report.py` — 巨大的硬编码模板字符串

从第 150 行开始的 90 行 markdown 模板直接内联在 Python 代码中。应使用 Jinja2 模板或通过 `importlib.resources` 读取外部模板文件。

## 🟠 模型与实验设计

### 9. 缺少学习率调度器

40 个 epoch 训练中使用固定学习率 1e-4。添加 `ReduceLROnPlateau` 或余弦退火可能提升最终性能，尤其在验证 loss 平台期时。

### 10. 缺少数据增强

医学影像去噪任务没有使用任何数据增强（翻转、旋转、轻微强度扰动）。2D slice 级别的增强（水平翻转、小角度旋转）可以提升泛化能力。

### 11. 缺少混合精度训练 (AMP)

`torch.cuda.amp` 可以在 RTX A6000 上显著加速训练（约 1.5-2x），且对去噪任务精度影响很小。

### 12. 缺少梯度裁剪

训练中未使用梯度裁剪。对于 U-Net 这类深层网络，`torch.nn.utils.clip_grad_norm_` 可以防止梯度爆炸。

### 13. 2D slice 独立处理丢失 3D 上下文

当前每个轴向切片独立处理。一种常见改进是 "2.5D" 方法：将相邻 3 个切片堆叠为 3 通道输入，或使用 3D U-Net（如果 GPU 显存允许）。

## 📊 训练数据分析

查看 [metrics.csv](runs/denoise_unet/metrics.csv)，训练曲线揭示了一些有趣的现象：

| 阶段 | Epoch | Train Loss | Val Loss | 观察 |
|------|-------|-----------|----------|------|
| 快速下降 | 1-12 | 0.00304→0.00266 | 0.00289→0.00270 | 最佳泛化点 |
| 平台期 | 13-36 | 0.00265→0.00254 | 0.00270~0.00276 | 过拟合开始 |
| 波动 | 36-40 | 0.00254→0.00254 | 0.00270→0.00275 | Val 不降反升 |

**关键发现**：训练在 epoch 12 后基本没有实际收益，36 epoch（epoch 36 的 val loss 0.00270 接近最佳）后的训练实属浪费。这进一步强调了第 1 点中早停的必要性。
